// schools

Schools are curated repositories of skills, conventions, and configuration for AI coding tools. Everyone on the same school shares the same skill set.

// structure

my-school/
├── school.toml      # school metadata & config
├── skills/          # shared AI skills
│   ├── commit.md
│   ├── review.md
│   └── ...
├── rules/           # coding conventions
│   ├── rust-coding.md
│   └── ...
└── CLAUDE.md        # session prompt template

school.toml defines the school name, services (GitHub, Linear), MCP servers, and project-specific overrides.

// create a school

$ mkdir my-school && cd my-school
$ git init
$ ace school init
# generates school.toml, skills/, rules/, CLAUDE.md

Add skills as markdown files in skills/. Add coding conventions in rules/. Push to a git remote.

// use a school

$ ace setup org/school # clone & configure
$ ace # skills are symlinked, backend launches

// propose changes

Skills are symlinked from the school cache into your project. Edits go directly to the cached school repo. To propose changes back:

$ ace diff # see what you changed
$ ace school propose # branch, commit, push, open PR
